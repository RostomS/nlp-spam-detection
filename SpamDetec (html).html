TP : Dtection de Spam dans des Messages Textuels

Dans ce TP, le but tait de russir dtecter si un message est un spam ou non, en se basant sur son contenu. Pour a, jai travaill sur un jeu de donnes contenant des SMS dj classs (spam ou ham), et jai construit un petit classifieur tape par tape.

On commencera par analyser et nettoyer les messages, puis on va extraire quelques caractristiques utiles (comme la longueur du message ou les mots suspects). Ensuite, on testera diffrents modles pour voir lequel serait le plus adapt : Naive Bayes, arbre de dcision, et random forest.

Tche 4.C valuation du modle :

Aprs avoir entran le modle Naive Bayes, jai valu ses performances sur les donnes de test.
Jai utilis plusieurs mtriques standards :

  • Accuracy : pourcentage total de bonnes prdictions.
  • Precision : parmi les messages prdits comme spam, combien sont rellement des spams.
  • Recall : parmi tous les vrais spams, combien ont t bien dtects.
  • F1-score : moyenne harmonique entre prcision et rappel (quilibre global).
  • Matrice de confusion : tableau des vraies vs fausses prdictions (utile pour bien visualiser les erreurs).
